# フロントエンド 詳細設計ガイド

フロントエンドの詳細設計で記録・検討すべき内容のガイドラインです。

---

## 設計の観点

### 1. コンポーネント設計

| 観点 | 内容 |
|------|------|
| コンポーネント分割 | ページ・機能・UIの3層で分割を検討する |
| 責務の明確化 | 各コンポーネントが単一の責務を持つよう設計する |
| Props設計 | 必須/任意Props、型定義を明確にする |
| 再利用性 | 汎用コンポーネントと機能特化コンポーネントを区別する |

### 2. 状態管理設計

| 観点 | 内容 |
|------|------|
| ローカル状態 | useState で管理するコンポーネント固有の状態 |
| グローバル状態 | Context / Zustand / Redux 等で管理する共有状態 |
| サーバー状態 | API取得データの管理（React Query / SWR 等）|
| 状態の正規化 | データの重複・不整合を防ぐ設計 |

### 3. API通信設計

| 観点 | 内容 |
|------|------|
| エンドポイント一覧 | 使用するAPIエンドポイントのリスト |
| リクエスト/レスポンス型 | 型定義の明確化 |
| エラーハンドリング | APIエラー時のUI表示・リトライ戦略 |
| ローディング状態 | 通信中のUI表現（スケルトン、スピナー等）|

### 4. ルーティング設計

| 観点 | 内容 |
|------|------|
| ページ構成 | URL構造とページコンポーネントの対応 |
| 認証ガード | 認証が必要なルートの保護方法 |
| パラメータ | URLパラメータ・クエリパラメータの設計 |

### 5. UI/UX設計

| 観点 | 内容 |
|------|------|
| レスポンシブ対応 | ブレークポイントと各デバイスでの表示方針 |
| アクセシビリティ | ARIA属性、キーボード操作対応 |
| エラー表示 | バリデーションエラー、システムエラーの表示方法 |

---

## 設計ドキュメントの保存場所

設計ドキュメントは以下の命名規則でこのディレクトリに保存する。

```
frontend/YYYYMMDD_機能名.md
例: frontend/20260414_user-login.md
```

テンプレートは [template.md](template.md) を使用してください。
