# フェーズ進捗ステータス

このファイルはウォーターフォール開発のフェーズ管理台帳です。  
**フェーズを開始・完了するたびに必ずこのファイルを更新すること。**

---

## ステータス定義

| ステータス | 意味 |
|-----------|------|
| `NOT_STARTED` | 未着手 |
| `IN_PROGRESS` | 進行中 |
| `COMPLETE` | 完了（次フェーズへの移行承認済み） |
| `BLOCKED` | 前工程未完了のためブロック中 |

---

## フェーズ一覧

| フェーズ | ステータス | 開始日 | 完了日 | 担当 |
|---------|-----------|--------|--------|------|
| フェーズ０: 基本設計 | `NOT_STARTED` | - | - | - |
| フェーズ①: 詳細設計 | `NOT_STARTED` | - | - | - |
| フェーズ②: 実装 | `NOT_STARTED` | - | - | - |
| フェーズ③: 単体テスト | `NOT_STARTED` | - | - | - |
| フェーズ④: 結合テスト | `NOT_STARTED` | - | - | - |

---

## フェーズ詳細

### フェーズ０: 基本設計

**ステータス**: `NOT_STARTED`

**開始条件**: なし（最初のフェーズ）

**完了チェックリスト**:
- [ ] 機能の目的・背景が明記されている
- [ ] 技術スタックとその選定理由が記載されている
- [ ] 却下した技術選択肢とその理由が記載されている
- [ ] システム全体のディレクトリ構成が示されている
- [ ] データモデルの概要が示されている
- [ ] APIエンドポイントの一覧（概要レベル）が示されている
- [ ] フロントエンドのコンポーネント構成が示されている
- [ ] 環境変数の管理方針が明記されている
- [ ] 実装フェーズ計画が示されている
- [ ] シャビの承認を得ている

**成果物**:
- `docs/artifacts/basic-design/<アプリ名>.md`

---

- [ ] API一覧が作成されている
- [ ] 各API処理設計書が作成されている
- [ ] エラーコード一覧が作成されている
- [ ] 画面遷移図が作成されている
- [ ] 画面設計書が作成されている
- [ ] シャビの承認を得ている

**成果物**:
- `docs/artifacts/detailed-design/backend/entity/`
- `docs/artifacts/detailed-design/backend/api/`
- `docs/artifacts/detailed-design/backend/error/`
- `docs/artifacts/detailed-design/frontend/screen/`

## 更新ルール

1. フェーズ開始時: ステータスを `IN_PROGRESS`、開始日を記入する
2. フェーズ完了時: チェックリストを全てチェック → ステータスを `COMPLETE`、完了日を記入 → シャビに報告して承認を得る
3. 前工程が `COMPLETE` でない場合、次フェーズは開始しない（`BLOCKED` と記録する）
