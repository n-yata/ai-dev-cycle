# フロントエンド 実装ガイド

フロントエンド実装の進め方と注意事項をまとめています。

---

## 実装の流れ

1. **コンポーネントの雛形作成** — ファイル・ディレクトリ構造を作成
2. **型定義の実装** — Props・State・APIレスポンスの型を定義
3. **UIの実装** — JSX/TSXでコンポーネントを構築
4. **ロジックの実装** — カスタムフック・ユーティリティ関数を実装
5. **API通信の実装** — データフェッチ・エラーハンドリングを実装
6. **スタイルの適用** — CSSモジュール / Tailwind 等でスタイリング
7. **セルフレビュー** — ガイドライン・セキュリティ観点で自己確認

---

## ディレクトリ設計の基本方針

```
src/
├── components/         # 再利用可能なUIコンポーネント
│   └── ui/             # ボタン、インプット等の基本部品
├── features/           # 機能単位のコンポーネント群
│   └── [feature-name]/
│       ├── components/ # 機能固有のコンポーネント
│       ├── hooks/      # カスタムフック
│       └── types/      # 型定義
├── hooks/              # アプリ共通のカスタムフック
├── lib/                # APIクライアント・ユーティリティ
├── pages/              # ページコンポーネント
└── types/              # 共通型定義
```
