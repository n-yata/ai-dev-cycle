# フェーズ④ 結合テスト（integration-test）

複数のモジュール・システム間の連携を検証するフェーズです。  
実際の依存関係（DB、外部API、ブラウザ等）を使い、システム全体の動作を確認します。

---

## このフェーズの目的

- フロントエンド・バックエンド間のインターフェースが正しく機能することを確認する
- データベースを含む実際のデータフローを検証する
- ユーザーが実際に操作する主要フローの品質を保証する

---

## 単体テストとの違い

| 観点 | 単体テスト | 結合テスト |
|------|----------|----------|
| 対象範囲 | 単一モジュール | 複数モジュール・システム間 |
| 依存関係 | モック化 | 実際の依存関係を使用 |
| 実行速度 | 高速（秒単位） | 低速（分単位） |
| 実行頻度 | コミットごと | PRマージ前・デプロイ前 |
| 目的 | ロジックの正確性 | 連携の正確性 |

---

## 実行環境

結合テストは専用のテスト環境で実行する。

- **DB**: テスト専用DBを使用（本番・開発DBとは分離）
- **外部API**: スタブサーバーまたはサンドボックス環境を使用
- **環境変数**: `.env.test` ファイルで管理

---

## 結合テストチェックリスト

- [ ] 主要なユーザーフロー（ゴールデンパス）がカバーされている
- [ ] 認証・認可フローが検証されている
- [ ] エラーケース（API失敗、ネットワークエラー）が検証されている
- [ ] データの永続化・取得が正しく機能している
- [ ] テスト後にデータがクリーンアップされている
