# フロントエンド 詳細設計ガイド

フロントエンドの詳細設計で記録・検討すべき内容のガイドラインです。

---

## 成果物

| 成果物 | テンプレート | 単位 |
|--------|-----------|------|
| 画面遷移図 | [screen-flow.template.md](screen-flow.template.md) | アプリ/機能群ごと |
| 画面設計書 | [screen-design.template.md](screen-design.template.md) | 画面ごと |

---

## 設計の観点

### 1. 画面遷移設計

| 観点 | 内容 |
|------|------|
| ページ構成 | URL 構造とページコンポーネントの対応 |
| 認証ガード | 認証が必要なルートの保護方法 |
| パラメータ | URL パラメータ・クエリパラメータの設計 |
| 遷移条件 | 各画面間の遷移トリガーと条件 |

### 2. コンポーネント設計

| 観点 | 内容 |
|------|------|
| コンポーネント分割 | ページ・機能・UI の3層で分割を検討する |
| 責務の明確化 | 各コンポーネントが単一の責務を持つよう設計する |
| Props 設計 | 必須/任意 Props、型定義を明確にする |
| 再利用性 | 汎用コンポーネントと機能特化コンポーネントを区別する |

### 3. 状態管理設計

| 観点 | 内容 |
|------|------|
| ローカル状態 | useState で管理するコンポーネント固有の状態 |
| グローバル状態 | Context / Zustand / Redux 等で管理する共有状態 |
| サーバー状態 | API 取得データの管理（React Query / SWR 等）|
| 状態の正規化 | データの重複・不整合を防ぐ設計 |

### 4. API 通信設計

| 観点 | 内容 |
|------|------|
| エンドポイント一覧 | 使用する API エンドポイントのリスト |
| リクエスト/レスポンス型 | 型定義の明確化 |
| エラーハンドリング | API エラー時の UI 表示・リトライ戦略 |
| ローディング状態 | 通信中の UI 表現（スケルトン、スピナー等）|

### 5. UI/UX 設計

| 観点 | 内容 |
|------|------|
| レスポンシブ対応 | ブレークポイントと各デバイスでの表示方針 |
| アクセシビリティ | ARIA 属性、キーボード操作対応 |
| エラー表示 | バリデーションエラー、システムエラーの表示方法 |

---

## 設計ドキュメントの保存場所

```
docs/artifacts/detailed-design/frontend/{成果物名}.md
例: docs/artifacts/detailed-design/frontend/screen-flow.md
例: docs/artifacts/detailed-design/frontend/login.md
```

設計書は継続的に更新する。ファイル内の「最終更新」で履歴を管理する。
