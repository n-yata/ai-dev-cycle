# バックエンド 実装ガイド

バックエンド実装の進め方と注意事項をまとめています。

---

## 実装の流れ

1. **型定義・インターフェースの作成** — リクエスト・レスポンス・ドメインモデルの型
2. **データモデル（スキーマ）の実装** — DBスキーマ・マイグレーションファイル
3. **リポジトリ層の実装** — DBアクセスロジック
4. **サービス層の実装** — ビジネスロジック
5. **コントローラー/ハンドラーの実装** — HTTPリクエスト処理
6. **ミドルウェアの適用** — 認証・バリデーション・ロギング
7. **セルフレビュー** — ガイドライン・セキュリティ観点で自己確認

---

## レイヤードアーキテクチャ

```
HTTPリクエスト
    │
    ▼
┌─────────────────────┐
│  Router / Handler   │  HTTPリクエストの受付・レスポンス返却
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  Middleware         │  認証・バリデーション・ロギング
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  Service Layer      │  ビジネスロジック
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  Repository Layer   │  DBアクセス
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  Database           │
└─────────────────────┘
```

---

## ディレクトリ設計の基本方針

```
src/
├── handlers/           # HTTPハンドラー（コントローラー）
├── services/           # ビジネスロジック
├── repositories/       # DBアクセス
├── middleware/         # 認証・バリデーション等
├── models/             # ドメインモデル・型定義
├── validators/         # 入力バリデーションスキーマ
└── utils/              # 共通ユーティリティ
```
