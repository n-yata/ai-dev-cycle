# テストパターン（test-patterns）

テスト実施で発見されたバグパターンや、効果的なテスト手法を記録するカテゴリです。  
「このバグはどのようなテストで防げたか」「このテスト手法が品質向上に効いた」という観点で蓄積します。

---

## このカテゴリに記録するもの

- 単体テスト・結合テストで発見されたバグのパターンと根本原因
- 「このテストを書いておけばよかった」という気づき
- 効果的だったテスト手法・モック戦略
- テストで防げなかったバグ（テスト設計の改善点）
- テスト実行速度・安定性の改善事例

---

## 記録のポイント

テストパターンの記録で最も重要なのは **「再現手順と防ぐためのテストコード」** です。

```typescript
// このバグを防ぐためのテスト
describe('パスワードリセットフロー', () => {
  it('期限切れのトークンを使うと401エラーになる', async () => {
    // バグ: 期限切れトークンでもリセットできてしまった
    const expiredToken = generateToken({ userId: '1', exp: Date.now() - 1000 });

    const response = await request(app)
      .post('/api/v1/auth/reset-password')
      .send({ token: expiredToken, newPassword: 'newPass123' });

    expect(response.status).toBe(401);
  });
});
```

## バグパターンの分類

| パターン | 説明 | 典型的な発見フェーズ |
|---------|------|-----------------|
| 境界値バグ | 0件、1件、最大値での挙動の違い | 単体テスト |
| 競合状態 | 非同期処理の順序に依存したバグ | 結合テスト |
| 状態の取り残し | テスト間のデータ汚染 | 結合テスト |
| 型の境界 | nullチェック漏れ、型変換の失敗 | 単体テスト |
| 環境差異 | ローカルでは動くがCIで失敗 | CI実行時 |
| タイムゾーン | 日時処理のタイムゾーン依存 | 単体テスト |
