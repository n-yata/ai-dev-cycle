## 概要

テスト実施後にテスト仕様書の状態・結果欄が「未着手・空白」のまま放置されていた。また、サブエージェントの完了報告を実際にテストを実行して確認せずに鵜呑みにしていた。この2点の反省から、ルールとして明文化した。

---

## 背景・コンテキスト

**状況:** フェーズ④（単体テスト）でギュレルがテストを実装・実行し「110件 Pass」と報告した。モドリッチはその報告をそのまま受け入れてコミット・完了フローに進もうとした。

**課題・問題:**
1. テスト仕様書3ファイルのステータスが「レビュー中」のまま、テストケースの状態・結果欄が全て「未着手・空白」だった
2. サブエージェントの報告を検証せずに「テスト完了」と扱っていた
3. シャビに「テスト実施完了している？」と指摘されて初めて気づいた

---

## 事実（何が起きたか / 何を決めたか）

- テスト仕様書3件のステータスが「レビュー中」→実際にテストが完了しているにもかかわらず更新されていなかった
- テストケース全行の「状態」「結果」欄が未記入のまま残っていた
- サブエージェント（ギュレル）がテスト実施後に仕様書を更新しなかった
- モドリッチがサブエージェントの報告を受けてから `npm run test` を実行して実際に確認すべきだったが、しなかった
- シャビの指摘後、実際に `npm run test` を実行して110件 Pass を確認し、仕様書を手動更新した
- ルールとして `docs/rules/4_unit-test/INSTRUCTIONS.md` に「1ケースずつ仕様書を更新する」ルールを追加した

---

## 理由・分析（なぜそうなったか / なぜそう決めたか）

### 根本原因

```
直接原因: サブエージェントがテスト実行と仕様書更新を別タスクとして認識し、仕様書更新を省略した
根本原因: テスト実施ルールに「仕様書の状態・結果を1ケースずつ更新する」という明示的な指示がなかった
```

---

## 次回への示唆

### すべきこと（Do）

- [ ] サブエージェントにテスト実施を依頼するときは「テストケース1件実行するごとに仕様書の状態・結果を更新すること」を明示的に指示に含める
- [ ] サブエージェントから「テスト完了・全 Pass」の報告を受けたら、モドリッチ自身が `npm run test`（または相当コマンド）を実行して実際に確認してからフェーズ完了フローに進む
- [ ] テスト完了フローに入る前に、仕様書の全テストケース行に「完了/OK」が記入されているか目視確認する

### 避けること（Don't）

- サブエージェントの「Pass した」という報告を実行確認なしに信頼する
- テストコードのコミット前に仕様書の状態確認をスキップする

### チェックリストへの追加提案

- [x] `docs/rules/4_unit-test/INSTRUCTIONS.md` に追記済み: 「1ケースずつ仕様書を即時更新」ルール
