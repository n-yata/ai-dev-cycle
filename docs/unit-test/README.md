# フェーズ③ 単体テスト（unit-test）

個々のコンポーネント・関数・モジュール単位の動作を検証するフェーズです。  
外部依存を最小限に抑え、対象の単位が正しく動作することを確認します。

---

## このフェーズの目的

- 実装した単位（コンポーネント・関数・サービス等）の正確性を保証する
- バグの早期発見とリグレッション防止
- リファクタリング時の安全網を提供する

## テスト作成の原則

### FIRST原則

| 原則 | 説明 |
|------|------|
| **F**ast | テストは高速に実行できる |
| **I**solated | テストは独立しており、他のテストに依存しない |
| **R**epeatable | 何度実行しても同じ結果になる |
| **S**elf-validating | テストの合否が明確に判定できる |
| **T**imely | 実装と同時（または直前）に書く |

### AAA パターン

```
// Arrange（準備）: テストデータ・モックをセットアップ
// Act（実行）: テスト対象の処理を実行
// Assert（検証）: 期待する結果を確認
```

---

## テストカバレッジの目安

| 層 | 目標カバレッジ | 重点対象 |
|----|-------------|---------|
| ユーティリティ関数 | 90%以上 | 全分岐 |
| サービス層 | 85%以上 | ビジネスロジック、エラーケース |
| コンポーネント | 70%以上 | 主要なインタラクション |
| ハンドラー/コントローラー | 80%以上 | 正常系・異常系 |

> カバレッジより「意味のあるテスト」を優先する。数値を達成するためだけのテストは書かない。

---

## テストしない対象

- サードパーティライブラリの内部動作
- フレームワークが保証している動作
- 単純なゲッター/セッター（ロジックがない場合）
